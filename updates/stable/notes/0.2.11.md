# Git Tool 0.2.11

## Features

- Branch context menus and commit graph branch/tag badges can now show a
  scoped first-parent history for the selected ref, with a clearable graph
  scope bar to return to all-branches history.
- Git Tool now quietly auto-fetches remotes while the app is unfocused, backing
  off during repository operations and applying the refreshed branch/graph state
  when the app regains focus.

## Fixes And Reliability

- Added the commit graph's missing `Revert commit to working tree...` context
  action so any right-clicked commit can be reverted into the working tree
  without creating a commit.
- Fixed checkout and sync for local branch names that contain slashes, so a
  branch such as `feature/demo` keeps using its configured upstream remote
  branch instead of being mistaken for a remote named `feature`. Push also
  preserves a configured non-`origin` upstream when its remote-tracking ref has
  been pruned as gone.
- Made manual merge review panes easier to copy from: single-click now focuses
  or selects text, double-click toggles a source line into the result, and the
  normal Monaco context menu is available again.
- Preserved the order users choose merge-conflict lines or side blocks in, so
  selecting incoming before current keeps that same order in the result file.
- Fixed manual conflict edits that deleted a blank line immediately around a
  resolved conflict block, including the Backspace path where the deleted
  newline sits between internal document segments.
- Fixed manual conflict edits where splitting a selected conflict line and then
  adding another source line could replace the custom manual text instead of
  appending the newly selected line.
- Fixed manual conflict review panes so visual blank padding used to align
  uneven ours/theirs blocks is no longer included when copying selected text.
- Added merge-commit smoke coverage that verifies the graph redraws after a
  conflict merge commit and the current branch shows its ahead-to-push
  indicator.
- Refreshed branch status, the graph, and working-tree state when a merge-class
  operation ends outside the app, so ahead indicators appear without requiring a
  manual fetch.
- Hardened merge conflict refreshes after applying AI/manual resolutions so
  files that Git has already staged at stage 0 do not reappear as phantom
  unresolved conflicts or drop their green resolved rows.
- Kept renamed applied conflict rows green after undo/reapply flows by matching
  applied decisions through working-tree rename/copy relationships.
- Backed ordinary staged resolved-conflict rows with Git's resolve-undo index
  metadata, keeping green rows and `Undo Apply` available after refreshes even
  when frontend applied-state tracking is stale.
- Pruned ordinary text-conflict applied markers from saved merge state once
  resolve-undo-backed status confirms Git can own the row, while preserving
  structural and submodule fallback markers.
- Routed merge conflict Apply, Undo Apply, structural Apply, accept/delete
  quick actions, and working-tree status streams through explicit repository
  paths so in-flight repo context changes cannot silently apply to the wrong
  repository or leave the right panel stuck on `Loading...`.
- Fixed working-tree sections that had rows available but still showed
  `Loading...` until the unstaged accordion was toggled.
- Fixed selected-row stashing from the working tree so unrelated staged files
  stay out of the new stash entry while remaining staged in the repository.
- Prevented duplicate synthetic applied-conflict rows from crashing the staged
  working-tree list when multiple saved conflict decisions point at the same
  resolved target file.
- Made the right-panel toggle recover from bad persisted/rendered panel state
  by explicitly showing the panel with a valid width instead of blindly
  flipping the stored boolean.
- Made in-progress `Resolve with AI` conflict requests cancelable from AI
  Assist, including cancelling the backend AI CLI process when possible. The
  active conflict header now stays as a `Resolving with AI` status/open action,
  with right-panel row spinners reserved for background requests.
- Added a global AI workflow timeout preset in Settings > AI > Workflows, and
  tightened structural-conflict prompting so hard compatibility mismatches
  return a reviewable related-target result instead of restoring obsolete
  original paths or spending the whole request budget on build-topology
  investigation. Same-name, same-method related files are treated as strong
  successor evidence even when a method gained an argument.
- Normal same-path text conflicts now use compact per-marker AI block
  resolutions when possible, with the backend composing the final file locally.
  This keeps large files with a few conflict blocks from timing out while the AI
  serializes thousands of unchanged lines, and Refine follows the same compact
  path for parseable conflicts. Those compact prompts now include deterministic
  package/import context and capped repository snippets for referenced
  method/type names before the AI runs.
- Normal same-path conflict review panes now show Git's true ours/theirs index
  stage blobs instead of marker-derived reconstructions, so the manual resolver
  matches full 3-way merge tools more closely while still applying through the
  editable result file.
- Normal same-path conflict review panes keep those stage blobs as full-file
  virtual revision views with original line numbers, so unchanged source
  content remains visible instead of being hidden behind separator rows.
- Merge conflict collection now batches unmerged stage blob reads through
  `git cat-file --batch`, so large conflict sets can open the merge workspace
  without paying one Git process per staged file version.
- Non-conflicting side-only changes in those full-file review panes now stay as
  normal, untinted source context, while actual conflict-zone lines remain the
  only highlighted and selectable source for manual resolution.
- Visible conflict ranges in the full-file review panes now get splitter
  connectors, making related ours/theirs blocks easier to track without
  inserting blank spacer bands.
- Full-file review panes now synchronize around the active conflict zone center
  instead of raw scroll pixels, keeping the zone checkbox centers aligned when
  one side has extra non-conflicting lines before the conflict.
- The result editor no longer paints whitespace-only conflict rows with the
  purple conflict tint or renders obsolete empty-anchor marker/spacer rows, so
  result files read as the text that will be applied.
- Structural related-target reviews now reserve purple result highlighting for
  actual manual conflict output instead of broad incoming evidence blocks.
- Structural AI now only chooses between `needs_review` and
  `no_related_target`. It can identify a related successor file for manual
  review, but it no longer declares changes already present, writes final
  content, restores originals, or deletes originals as AI result states.
- AI workflow settings now flag the conflict-resolution provider safety posture
  in the readiness tooltip, keeping each workflow row to one visible status
  badge, and AI Assist refine includes one-click instruction starters for common
  correction requests.
- AI workflow model, timeout, and writing-skill changes now auto-save with
  compact saving/saved feedback instead of a separate Save AI Settings button.
- Structural AI now receives a backend-generated base-to-modifying-side diff
  for delete/modify conflicts and treats it as the primary change list, so
  small mechanical changes are less likely to become giant synthetic reviews
  against broad refactor noise.
- Structural related-target review now opens a manual synthetic conflict against
  the successor, rather than attempting to pre-port simple insertions or clean
  replacements automatically.
- Structural related-target results now always open a manual synthetic review
  when a separate successor file is found. The old ready-to-apply structural AI
  result screens were removed so no stale "already present" or AI-written
  content pane can appear.
- Structural related-file comparisons now use the same full-file source-backed
  review panes as manual conflict review instead of Monaco's generic
  side-by-side diff editor, removing the large blank alignment blocks from
  structural evidence and synthetic review surfaces.
- Structural related-file review now uses Git's histogram diff hunks to choose
  synthetic conflict zones, reducing extra checkboxes and odd hunk splits from
  the previous frontend line matcher.
- Structural related-file review now coalesces noisy Git hunk sets by retrying
  with larger inter-hunk context when the initial synthetic review would create
  too many zones.
- Identical structural related-file reviews no longer manufacture a whole-file
  conflict block when Git reports no synthetic diff hunks.
- Markerless structural related-file reviews keep source-specific pane labels
  instead of showing `Merged Result` on both sides.
- The manual result pane now highlights unresolved empty conflict anchors on
  their insertion line, so normal conflicts still show where empty zones land
  without reintroducing fake spacer rows.
- Structural related-file review now uses the same source-backed review,
  empty-side widening, manual document, and zone-selection flow as ordinary
  same-path text conflicts after the backend has generated those Git hunk
  zones.
- Normal conflicts and structural related-file reviews now pass through a
  normalized review session before rendering, leaving the remaining
  structural-specific behavior at source loading and apply-target routing.
- Low-confidence AI-authored conflict resolutions now still prefill the result
  pane when usable resolved content is returned, while keeping the warning in AI
  Assist so the user can review the uncertain parts.
- AI Assist now opens before the first conflict-resolution run, letting users
  add optional guidance before clicking `Resolve with AI`; the same panel then
  hosts summary, refine, and re-generate actions.
- Normalized absolute structural AI target paths back to repository-relative
  paths before reading, reviewing, or applying them, preventing duplicated
  `/repo//absolute/path` read failures.
- Improved structural related-file scoring with normalized source-content
  similarity, so mostly identical current files now rank as strong successor
  candidates instead of depending only on basename, source-set, and symbol
  signals.
- Structural related-file discovery now also scans same-extension current files
  for shared path terms, symbols, literals, and source similarity, so renamed
  successors such as `ExportHandler.java` -> `ExportController.java` can be
  offered to AI review even when the basename changed.
- Improved generated AI text formatting: conflict AI now asks for structured
  explanation detail rows instead of numbered prose blobs, AI Assist renders
  those rows with labels, and branch/commit AI result dialogs render sanitized
  Markdown.
- Swapped the AI Assist summary copy pill for the same compact copy/check icon
  affordance used by the conflict header path copy action.
- Matched the in-progress merge header background to the unselected conflict
  file row gold so the conflict state reads consistently across panels.
- Hid the right-panel commit box while merge-class conflict operations are
  still unresolved, bringing it back only once the operation is ready to
  commit or continue.
- Kept the conflict navigation arrows usable for `1 / 1` files, so either
  arrow can jump back to the only conflict after scrolling away.
- Moved manual and structural conflict `Apply` actions into the conflict header
  beside the AI action, removing extra bottom apply docks from review
  workspaces.
- Gave the conflict header path more spacing from action buttons and made it
  clickable to copy the full file path with hover and copied feedback.
- Made AI Assist re-generate/refine actions show clear in-progress states, with
  re-generate matching the main cancelable `Resolve with AI` button and the
  conflict header showing `Refining...` or `Regenerating...` only while AI
  Assist is closed.
- Made AI Assist refinements cancelable through the same backend AI cancel path
  as initial conflict resolution, and disabled re-generate while a refinement
  is in progress.
- Stopped background AI conflict results from auto-opening AI Assist over the
  currently displayed conflict; only the active conflict's completed request can
  reveal the panel.
- Refined the AI Assist panel styling with a purple AI accent, compact
  AI Suggestion header actions, and a composer-style refinement area.
