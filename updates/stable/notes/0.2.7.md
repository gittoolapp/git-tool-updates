# Git Tool 0.2.7

## Repository Workflows

- Working-tree and commit-detail context menus now include the expected file
  actions: `Show in Finder`, `Copy file path`, `File History`, and `File Blame`
  where they apply. Multi-file selections can copy all absolute paths at once.
- Sidebar tags are now selectable and open the tagged commit in the history
  inspector.
- Sidebar stashes now show a muted compact creation-age label on the right side
  of each row.
- Sidebar tags now have a right-click menu for checkout, branch creation,
  merging the tag into the current branch, cherry-pick, reset, local/remote tag
  push/delete, copy actions, and supported provider tag links.
- Stash details can restore a single file without applying or dropping the rest
  of the stash, while still refusing to overwrite dirty files.
- Stashing selected working-tree rows from their context menu now refreshes the
  sidebar stash list immediately.
- The shared file inspector now has a workspace-style header with file-level
  `Stage File` and `Unstage File` actions for working-tree selections, plus a
  dedicated second-row toolbar for view and diff controls.
- File inspector diffs can now switch between the default unified hunk view and
  a side-by-side split view, with previous/next hunk navigation and active hunk
  highlighting available in the header.
- The shared file inspector header aligns with the sidebar filter area, and
  unified hunk headers tint through the line-number gutter.
- Markdown files in the shared file inspector now expose a sanitized read-only
  `Preview` mode next to `Diff` and `File`.
- Unstaged working-tree files can now be edited directly from the shared file
  inspector and saved back to the working directory.
- Fetch, push, and remote actions now disable earlier when a repository has no
  remotes or still points at a missing remote, with clearer blocked states and
  tooltips.
- Commit graph `Cherry-pick commit...` and `Revert commit...` actions now
  disable before confirmation until the repository's assigned account has a
  commit name and email.
- Commit graph edit-message/drop history rewrites now keep the checked-out
  branch checked out for non-tip commits, and the edit dialog no longer
  overwrites fast user edits when the full commit message finishes loading.

## Pull Requests

- Pull-request commit rows are now selectable, showing the selected commit's
  changed files, copy actions, and center-panel diffs in the review workspace.
- Pull-request `Files` / `Commits` tabs now use the compact boxed segmented
  style shared with the file inspector.
- The pull-request workspace now opens in a focused layout by temporarily
  collapsing the left sidebar.
- GitHub SSO and expired-token warnings now include an `Open Accounts` shortcut.

## Fixes And Reliability

- Manual ref-to-ref release patch creation now reuses existing source and
  target artifacts when available, builds missing source artifacts in a
  temporary workspace, and only writes a target artifact when it is missing, so
  publishing a new patch no longer replaces the prior release artifact or
  changes its checksum files.
- Merge and rebase completion now stay available when the final conflict
  resolution leaves no visible staged diff, and resolved conflict rows keep the
  `Undo` action available.
- Merge-ready toolbar status stays on one line at narrow widths instead of
  growing into a crowded multi-line badge.
- Startup update checks now wait until the initial environment and recent
  repository bootstrap has finished, so the title-bar update badge appears
  promptly instead of waiting for Settings to run a manual check.
- Context menus now open in the app-level overlay layer, so graph, tag, stash,
  and other right-click menus stay aligned to the click and are not clipped by
  sidebar or graph panel scroll containers.
- Monaco inspectors now repaint and relayout correctly when opened, restored, or
  switched between diff, file, blame, and history modes.
- Manual conflict edits preserve the rest of the selected conflict block, and
  single-file discard/delete actions no longer close unrelated inspector diffs.
- Manual and AI conflict Apply now preserve final newlines and trailing blank
  EOF lines from the conflicted file.
- Native window size and position now persist across more app shutdown paths.
- Added broader Playwright smoke coverage for repository, branch, merge, rebase,
  pull-request, stash, submodule, and inspector workflows.
