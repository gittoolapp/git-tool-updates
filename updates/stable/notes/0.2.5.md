# Git Tool 0.2.5

## Branch Workflows

- Added `Rebase current branch onto this...` to the local branch context menu
  so the checked-out branch can be rebased onto another local branch without
  leaving the sidebar workflow.
- Added `Force Push with Lease...` to the local branch context menu so
  intentionally rebased or amended branch history can be published without
  leaving the app's guarded branch workflow.
- Reorganized local branch context menu remote actions into smaller sync,
  publish, and pull-request groups so rebasing-related actions are easier to
  scan.
- Sidebar branch dialogs now dismiss on `Escape`, including force-push,
  upstream selection, rebase confirmation, and AI branch explanation modals.
- Simplified the force-push and rebase confirmation dialog headers by removing
  the redundant branch label from the top-right corner.
- Rebases that stop for conflicts now reuse the existing conflict workspace,
  abort flow, and working-tree completion guardrails, including continue/abort
  support through the normal operation UI.
- When a rebase is ready to proceed, the staging panel now offers
  `Continue rebase` directly instead of forcing a manual commit message path.
- Text selection highlights now stay visible in conflict editors even when the
  selected text crosses tinted conflict-zone lines.
- Cmd/Ctrl+F now opens Monaco search for the active file or diff inspector,
  even when focus is on the inspector chrome instead of the editor text.
- Working-tree Monaco diffs now show per-hunk actions in the hunk header:
  unstaged hunks can be discarded or staged, and staged hunks can be unstaged
  back out of the index without touching the working-tree edit.
- Added/deleted file hunk actions now emit Git file-mode patch headers so
  staging a newly-created file hunk does not try to stage a bogus `dev/null`
  path.

## Reliability

- Guarded merge completion through the normal Commit button so resolved files
  that were edited again must be staged or discarded before the merge commit is
  created, preventing those edits from being left behind as surprise unstaged
  changes.
- Reverting a merge commit now uses the first parent as the mainline, matching
  the app's current-branch history view and undoing the merged-in changes
  instead of surfacing Git's raw `no -m option was given` failure.
