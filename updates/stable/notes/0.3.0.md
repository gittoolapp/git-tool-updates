# Git Tool 0.3.0

## Highlights

- Large repositories should feel smoother: staging rows, branch lists, commit
  history, and file history now render with windowing, while file diffs use
  caching and nearby-file prefetching to make repeated navigation faster.
- Terminal sessions now stay alive when the terminal panel is hidden. Terminal
  tabs can be renamed, closed with `Cmd/Ctrl+W`, opened with `Cmd/Ctrl+T`, and
  scrolled horizontally when many sessions are open.
- Repository actions now use short-lived success and error toasts, with durable
  details and repair actions kept in Error Center.
- The title bar now includes Activity history for recent Git-impacting actions,
  including the repository, target, timestamp, and success or error result.
- Commit-message drafts now persist per repository and branch, including AI
  generated drafts, and are restored after switching repositories or restarting
  the app.
- The commit graph can show first-parent history for a selected ref, display
  multiple branch/tag badges with overflow handling, checkout branches by
  double-click, and highlight detached HEAD commits.
- Manual fetches now show newly discovered remote branches in a temporary
  `Recently fetched` group, while quiet background fetches refresh branch and
  graph state when the app regains focus.

## Workflow Improvements

- File inspectors now keep the selected file path as the main header detail:
  commit file views no longer repeat a `COMMIT <hash>` pill, unstaged diffs use
  an `UNSTAGED` label, and header paths can be clicked to copy.
- Settings and onboarding layouts are more stable at different window sizes,
  with better scrolling for account, SSH key, repository access, and AI setup
  panels.
- AI workflow settings now auto-save model, timeout, and writing-skill changes
  with compact saving feedback instead of a separate save button.
- AI readiness tooltips now explain the conflict-resolution provider safety
  posture without adding extra visual noise to each workflow row.
- AI generated branch, commit, and rewrite explanations now render Markdown in
  the result dialog while preserving the original text for copy actions.
- Pull request, commit, and setup blockers now route repair actions through
  toasts and Error Center so the current workspace stays focused.

## Conflict Resolution

- Same-path text conflicts now show Git's true ours/theirs stage versions as
  full-file review panes with original line numbers, visible unchanged context,
  and conflict-zone highlighting only where the file actually conflicts.
- Manual conflict review is easier to use: source panes are copyable again,
  selected lines keep the order the user chose, and blank alignment padding is
  no longer copied as source text.
- Conflict apply actions now live in the conflict header beside AI actions, and
  conflict file paths can be copied directly from the header.
- `Resolve with AI`, re-generate, and refine flows now have clearer in-progress
  states and can cancel the backend AI process when possible.
- AI conflict resolution now uses compact per-conflict-block prompts for normal
  text conflicts when possible, reducing timeouts on large files with small
  conflict regions.
- Structural conflict AI now stays review-first: it can suggest a related
  successor file for manual review, but it no longer writes final content,
  restores deleted originals, or declares a structural change complete on its
  own.
- Structural related-file reviews now use Git histogram hunks and the same
  source-backed manual review experience as normal text conflicts, reducing
  noisy synthetic conflict zones and oversized blank diff areas.

## Fixes And Polish

- Disabling local SSH-agent use now also prevents fetch, pull, push, and remote
  tests from falling back to the user's default SSH agent or SSH config when no
  repository SSH key is pinned.
- Branches with slashes now keep checkout, sync, and push behavior tied to the
  correct upstream remote branch, including non-`origin` upstreams.
- Local branches are ordered by recent checkout history, with the current
  branch first and never-used branches alphabetized.
- Loading older commit graph history keeps the viewport in place instead of
  snapping back to the newest commits.
- Pull actions now bring the newly pulled branch tip back into view in the
  commit graph, and AI writing-skill resets immediately return the workflow to
  its default skill state.
- Pull request setup warnings now show their matching repair action directly in
  the workspace when account or repository settings need attention.
- Working-tree diffs now stay on the right file after staging or unstaging,
  including moved files and partially staged files.
- Rename-plus-edit rows now show rename context only on the matching staged or
  unstaged row, so the remaining edit appears under the new path.
- Stashing selected working-tree rows no longer pulls unrelated staged files
  into the stash.
- Conflict refreshes after AI/manual apply are more reliable: resolved rows stay
  green, undo remains available where Git has resolve-undo data, and stale
  repository switches no longer apply conflict actions to the wrong repository.
- The right-panel toggle now recovers from bad persisted panel state by showing
  the panel with a valid width.
- Rounded pill styling has been replaced by squared shared radius tokens across
  controls, badges, tabs, cards, popovers, modal surfaces, and compact
  indicators so the app reads as one visual system while loading/action spinners
  remain circular.
