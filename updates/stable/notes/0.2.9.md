# Git Tool 0.2.9

## Features

- Added Settings > Appearance themes with curated dark and light palettes,
  including Graphite, Bright Dark, High Contrast Dark, Mist Light, and High
  Contrast Light. Theme changes apply immediately and persist app-wide across
  environments, and each option shows a small preview built from the real theme
  tokens. Light themes use darker text tokens, Monaco's light syntax palette,
  and dedicated graph lane colors. Version and update controls now live
  separately in Settings > Version & Updates.
- Added the first slice of guided onboarding for the no-repository state. First
  run now shows a skippable setup path for Repository, Identity, AI, and Start
  while keeping open repository actions immediately available.
- Opening a repository from first-run setup now keeps the user in setup and
  advances directly to Identity, so SSH keys, account, and provider setup are
  the first actionable follow-up instead of a separate readiness screen.
- Added focused Settings-based Identity and AI steps to repository setup. The
  setup page now reuses the real account, repository access, credentials, and
  AI panels while keeping unrelated settings unavailable, uses `Skip setup` as
  the single workspace escape hatch, and avoids a multi-card setup dashboard.
- Locked repository setup into a guided sequence so SSH Keys, Accounts,
  Repository Access, AI, and Start advance with `Next` and `Previous` instead
  of letting users jump around setup pages.
- Replaced the setup-mode Settings sidebar with a compact Identity sub-step
  progress row, keeping AI as the next major setup step instead of a nested
  setup task.
- Tightened the repository setup layout and expanded the setup frame so
  onboarding uses more of the available workspace before inner panels scroll.
- Bounded the repository setup frame horizontally while keeping it vertically
  stretched, so wide windows keep the guided setup surface readable instead of
  stretching edge to edge.
- Kept repository setup contained inside the available center panel at compact
  window sizes so the outer onboarding workspace does not show its own
  scrollbar.
- Trimmed setup-mode Settings panels so onboarding hides workflow model tuning,
  advanced credential helpers, technical repository details, and destructive
  account/auth/remote cleanup controls while full Settings keeps them
  available. AI setup still shows every supported provider with install
  guidance for missing CLIs.
- Fixed Known Hosts table alignment, kept successfully trusted custom hosts
  visible after the credential status refresh, and added removal for custom
  host entries.
- Added a title-bar `Resume setup` action after setup is skipped from an open
  repository, so users can return to missing setup steps later.
- Onboarding progress now survives app restart per environment without forcing
  setup onto profiles that already have repositories. Restored repositories open
  the normal workspace, with `Resume setup` available when setup was unfinished
  or skipped.
- Added the final onboarding Start step, so AI setup can advance into a clear
  `Open workspace` finish action that marks setup complete instead of leaving
  it resumable.
- Added a first-run review step for `Open Repositories Folder`. Folder imports
  now scan first, group discovered repositories with readiness labels, and save
  only the selected repositories after `Import Selected`.
- Expanded folder-import readiness so each scanned repository reports commit
  identity, remote, and access checks before import, including suggested
  matching accounts without silently assigning them.
- Added the first identity-rules slice. Trusted rule matches now stay as
  suggestions during single-repository open/restore so onboarding never selects
  an account before confirmation; selected folder import can still persist the
  planned assignments after the import confirmation step, while the folder scan
  stays read-only and shows those plans before import.
- Folder import review now supports assigning one account to the selected
  repositories in a group. Import persists those Git Tool assignments and can
  seed a reusable remote-owner rule without writing repository-local SSH or
  commit identity config.
- Repository Access now reconciles accountless folder imports after onboarding
  users create accounts. Repositories without a Git Tool account are shown in
  an Unassigned Repositories group surface, can suggest one clear account, and
  can be assigned as a group without writing repository-local Git config.
- Repository Access now treats bulk group assignment as an explicit saved
  intent, not an access proof. The unassigned-groups surface leads the page
  when work remains, labels assignments as access not tested, shows suggested
  accounts as their own chip beside the selector, explains saved-rule vs
  matching-provider suggestions, and avoids provider access prompts until a
  matching repository remote is actually recognized. In full Settings, a scope
  switch now separates All repositories cleanup from the Current repository
  setup surface, stays visible while the assignment queue is still loading, and
  reuses a backend cache until recents, account assignments, accounts, identity
  rules, or repository Git config timestamps change.
- Repository Access setup mode now hides the full diagnostic sections and remote
  inventory. During onboarding it shows only the active assignment task:
  unresolved repository groups for folder imports, an account-required task
  before any accounts exist, or a compact current-repository assignment surface
  for single-repository setup. Suggested single-repository accounts are marked
  inside the account button, ambiguous choices show a short explanation, and
  Test Access is hidden until an account is assigned. After a single repository
  account is selected, the compact surface switches to an assigned state and
  keeps only the remaining actions visible.
  Folder-import setup now shows a repository assignments complete state after
  the final group is assigned. It also shows a loading state while imported
  repository groups are first read, then keeps existing group rows stable during
  background refreshes. The setup source is preserved across repository loading
  and resume state, so folder imports stay on the grouped assignment path while
  single-repository setup stays compact. Switching profiles into an environment
  with setup still in progress now restores the setup workspace directly instead
  of showing the normal repository workspace with only a resume button, and app
  restart does the same. The active setup stage is also saved per environment,
  so returning to onboarding reopens the last guided stage instead of resetting
  Identity to the first panel.
- Explicit repository account assignment now seeds a reusable remote-owner
  identity rule when the repository has a recognized remote owner, so future
  repositories under the same owner can be assigned automatically without
  relying on weak single-account guesses.
- Repository Access > All repositories now includes trusted auto-assignment
  rules beside the assignment queue, so rules can be reviewed, paused, or
  deleted without a separate Settings section.
- Repository Access now refreshes the embedded automatic assignment rules after
  single-repository or group assignment, so newly seeded remote-owner rules show
  up without manually refreshing the rules panel.
- The embedded rules list now backfills missing remote-owner rules from existing
  confirmed repository assignments, so rules still appear if the assignment
  queue was already cleared before the rules surface refreshed.
- Automatic Assignment Rules now reuse a backend cache and keep the last rules
  visible while refreshing, avoiding the empty reload flash when reopening
  Repository Access with unchanged repository/account state.
- Added explicit SSH vs HTTPS Git access modes for accounts. HTTPS accounts no
  longer require an SSH key, repository assignment clears only app-managed SSH
  pins, readiness calls out transport mismatches, and provider clone uses the
  HTTPS URL when the selected account relies on Git credential-helper or
  token-backed credentials.
- Clearing repository auth and deleting the assigned account now preserve
  unrelated custom `core.sshCommand` values, while still removing Git Tool's
  own managed SSH pin.
- Added one-click remote transport repair for SSH/HTTPS mismatches. When the
  assigned account mode and a convertible remote differ, the remote row can
  switch the URL to the matching transport without requiring manual edit.

## Fixes And Reliability

- Prefilled the commit message for completed merge-conflict resolutions with
  the merge branch direction, while preserving user edits.
- Added the missing right-panel close action for selected stash details, matching
  the commit-detail return-to-working-tree behavior.
- Cleared the shared sidebar filter after successful branch checkout so later
  tag, stash, and submodule browsing is not hidden by a stale search.
- Reset the sidebar sections to simple header-collapsible panels, removed the
  broken remembered-height accordion persistence, and restored lightweight
  resize handles between neighboring sections, including collapsed headers next
  to an expanded section; the last expanded section now fills leftover sidebar
  height instead of keeping an unnecessary inner scrollbar, and expanded
  sections keep a taller minimum height so a few rows remain visible.
- Let oversized sidebar section layouts compress expanded sections back toward
  their minimum heights before showing a whole-sidebar scrollbar.
- Kept sidebar section separators mounted while sections collapse or expand, so
  collapsed headers below an opened section no longer nudge by a few pixels.
- Added a visible sidebar header close button and widened the sidebar resize
  rail so closing and dragging the left panel match the right panel's shell
  behavior.
- Strengthened the selected commit color and same-size graph dot so selecting a
  branch tip from the sidebar is easier to see in the commit graph.
- Added branch actions to commit-graph branch badges, so right-clicking a graph
  branch label can checkout, merge in either applicable direction, or copy the
  branch name without returning to the sidebar.
- Suppressed the native focus rectangle on commit-graph branch badges after
  opening their context menu and closing it with Escape.
- Added incremental older-history loading to the commit graph: scrolling near
  the bottom of a full 1000-commit window requests the next 1000 commits instead
  of stopping at the initial graph window.
- Selecting a branch whose tip is outside the loaded commit graph now requests
  older history until the tip is available, then scrolls the virtualized graph
  viewport to it.
- Selected the definitive pulled branch tip after successful pulls so the
  commit graph viewport moves to the branch's updated location instead of only
  updating the commit detail panel.
- Added a general compare workspace for branch-to-branch, branch-to-commit, and
  commit-to-commit reviews, available from branch and commit graph context
  menus with shared file and commit inspectors. General compare now uses a
  direct ref-to-ref diff so branch tips that differ still show changed files
  even when one selected ref is an ancestor of the other.
- Grouped compare context-menu actions into their own section and strengthened
  menu divider contrast so dark-theme menu groups are easier to scan.
- Renamed the local branch upstream action to `Set Remote Branch...` or
  `Change Remote Branch...`, making the existing tracking-branch picker match
  the way users describe changing a local branch's remote branch.
- Added Settings > Shortcuts with search, capture, and reset controls for
  app-level shortcuts, including configurable defaults for Find, repository
  finder, terminal toggle, fetch, pull, push, branch creation, pull request,
  stash, and pop stash actions.
- Added multiple embedded terminal sessions with tab switching, `+` creation,
  and per-terminal close controls.
- Polished the embedded terminal tab strip so terminal tabs start from the left
  edge, the repeated repository path stays out of the visible chrome, and zsh
  opens without its startup partial-line `%` marker.
- Made row and toolbar hover states use the themed interactive hover surface so
  branch, file, settings, graph, error-center, and conflict controls still
  visibly highlight in light themes.
- Made dense file, branch, tag, stash, and commit-file row hover backgrounds
  respond immediately instead of fading, so quick pointer movement no longer
  feels delayed or leaves a hover afterimage.
- Extended immediate hover feedback to other scan-through surfaces, including
  repository and environment pickers, context menus, settings directory rows,
  pull-request result rows, and branch upstream choices.
- Made already loaded Monaco editors switch their base light/dark theme
  immediately when the app theme changes, preventing a brief opposite-theme
  flash when opening files after toggling Appearance.
- Closed Settings automatically after creating or duplicating an environment, so
  the app lands directly in the newly activated workspace.
- Hid the empty repository sidebar in the no-repository onboarding state, giving
  first-run setup the full workspace.
- Kept restored existing environments out of onboarding mode, so previously
  configured profiles reopen directly into the normal workspace.
- Stopped repository open from auto-assigning the only configured account.
  Repository-local SSH and commit identity settings now wait for an explicit
  account assignment or a trusted identity-rule match.
- Deleting an account now also removes identity rules that point at that
  account, preventing stale future auto-assignment hints.
- Updated the first-run setup header so its title and description follow the
  active Identity, AI, or Start step after using `Next`.
- Replaced native blue focus rings on custom sidebar/repository rows with app
  focus surfaces, so closing a context menu with Escape no longer leaves a
  rectangular browser outline around stash, tag, branch, or repository rows.
- Fixed shared floating popovers so async-loaded dropdown content can resize
  after it arrives, preventing the repository switcher from sometimes showing
  only the first recent repository.
