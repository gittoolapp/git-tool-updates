# Git Tool 0.2.0

## Highlights

- Redesigned the Settings experience with clearer navigation, denser account and
  key management, a stronger AI setup panel, and more focused Repository Access
  and Environments pages.
- Expanded branch and commit context menus with copy actions, upstream setup,
  tag creation, guarded checkout/reset/save-patch actions, guarded commit
  edit/drop actions, and preview-first AI explanations and rewrite suggestions.
- Improved merge and conflict resolution with clearer file actions, more
  reliable applied-state tracking, sturdier `Undo apply` behavior, and safer
  structural AI resolution flows.
- Made the commit graph smoother during live resizing and reduced unnecessary
  graph reloads when repository topology has not changed.

## Settings And Updates

- Made update checks quieter and more persistent in Settings, with always-on
  background checks and automatic checks when the Application page opens.
- Added internal update installation: verified downloads can now use **Install
  and Restart**, which prepares the cached patch/full artifact, launches a
  copied helper, preserves rollback, and restarts Git Tool without asking the
  user to manage patch files.
- Removed the separate Background Checks setting and the pre-download warning
  from available updates.
- Tightened the title-bar update badge and refreshed Settings surfaces,
  disabled states, focus states, account rows, SSH key rows, known-host rows,
  and public-key previews.
- Custom AI workflow models now show their saved model name in selectors, with
  clearer validation and save feedback.
- Added dedicated AI workflow settings for branch explanations, commit
  explanations, and commit rewrite previews.

## Repository Workflows

- Detached HEAD now appears as a selected `HEAD` row in LOCAL, with limited
  actions to create a branch or copy the commit SHA.
- Added branch upstream selection, explicit lightweight and annotated tag
  creation, and push-then-create-pull-request from branch actions.
- Added guarded commit checkout, reset-to-commit, save-patch, edit-message, and
  drop actions from commit context menus.
- Added context-menu copy actions for branch names, branch-tip SHAs, commit
  SHAs, and supported remote commit links.
- Improved long context menus so they stay inside the app window and scroll only
  when needed, and restyled context menus and action dialogs to match Settings.
- Suspended app-level keyboard shortcuts while modals or context menus are open.

## Staging And History

- Made the staging commit message input resizable upward and persisted its
  height across restarts.
- Made stage, unstage, and conflict-apply actions update row placement and focus
  more reliably while working-tree refreshes catch up.
- Fixed staging and shared file-list path labels so long paths use available
  row width before truncating middle folders.
- Rendered commit graph lanes and dots through one virtualized overlay layer,
  improving resize performance.
- Kept commit graph ordering stable by avoiding stale branch-tip promotion, and
  skipped preserve reloads when refs and HEAD are unchanged.
- Fixed post-commit and post-push graph refreshes so newly created current-branch
  commits do not stay visually anchored in older history until an app reload.

## Merge And Conflict Resolution

- Fixed merge-complete transitions so the center panel reliably returns to graph
  mode without toolbar or empty-graph flashes.
- Kept applied conflict resolutions durable across restart, including staged
  resolved rows and `Undo apply` from the working tree.
- Simplified applied conflict state into one resolved-decision model shared by
  text, delete, structural, and submodule resolutions.
- Moved applied-resolution controls out of the center conflict card so staged
  rows are the durable home for undoing an applied resolution.
- Improved `Undo apply` so restored conflicts regain focus immediately and avoid
  brief generic conflict-editor flashes.
- Refined conflict-card headers, inline text-conflict review, structural
  conflict review, AI Assist summaries, and active-merge warning surfaces.
- Restored text selection inside conflict review and manual resolution editors
  while keeping editor chrome non-selectable.

## AI Conflict Resolution

- Improved structural AI results with explicit operations for using existing
  target files, writing content, deleting originals, or restoring originals.
- Added safeguards for deletion recommendations so risky or low-confidence
  structural resolutions route back through review instead of producing empty
  results.
- Ranked related-file candidates with backend evidence such as source set,
  module or namespace overlap, shared symbols, and distinctive text literals.
- Bounded AI-visible related-file content with outlines and omitted-line markers
  while keeping full file bodies available in the review UI.
- Reduced structural-result flashes by preparing related-file comparison
  workspaces before opening them and by keeping transition states blank during
  apply and undo refreshes.
