# Git Tool 0.2.3

## UI Polish

- Removed the visible borders from commit graph branch and tag reference chips,
  made the chips taller, and set the commit graph row height to 30px so row text
  and lane markers read larger.
- Grouped remote branches under collapsible remote-name rows in the sidebar so
  branch labels no longer repeat prefixes like `origin/`.
- Added a remote-folder context menu with fetch, manage, open-on-provider, and
  copy-remote-link actions.
- Expanded remote branch context menus with existing checkout, create local
  branch, merge, fetch, AI explain, copy name/SHA, and remote commit-link
  actions.
- Let branch ahead/behind badges expand for larger counts instead of clipping
  inside the sidebar row.
- Replaced the commit-message AI sparkle glyph with a clearer icon treatment
  and added inline feedback when commit or AI setup blocks the action.
- Added a close action to the Commit Detail panel header so users can return
  directly to the Working Tree panel.
- Refreshed the sidebar tag list after creating tags from commit history, so
  new tag chips in the graph also appear in the TAGS section.
- Centered AI provider initials and workflow numbers inside their circular
  Settings marks.

## Settings And Updates

- Changed available updates to a single **Install and Restart** action that
  downloads, verifies, prepares, installs, and restarts internally without
  exposing a separate patch-download step.
- Removed patch-file management from the normal Settings flow and clears the
  cached patch/full artifact from the update cache after the staged install
  payload is prepared.
- Fixed release patch generation so the default app-managed update path fails
  during packaging if `bsdiff` is missing instead of publishing an unsupported
  `xdelta3` patch.

## AI And Repository Workflows

- Added Gemini CLI as a supported AI provider alongside Codex and Claude.
- Updated Gemini presets to include the current Gemini 3 text models while
  keeping Gemini 2.5 options as stable fallbacks.
- Empty AI workflow model settings now default to a model compatible with an
  installed supported CLI before falling back to the built-in Codex default.
- Added a labeled **Fetch** action to the main repository toolbar.
- Added a RepoSwitcher remove action for clearing repositories from the current
  environment's recent list.
- GitHub PR readiness no longer clears a connected token when GitHub returns a
  repository permission, scope, or organization SSO `403`; it keeps the token
  and shows permission guidance instead.
- Removed the duplicate pull request readiness message from the Details footer;
  the Target card remains the single visible status location.
- Repository Access now treats provider-backed pull request features as optional
  readiness, so GitLab, Bitbucket, and basic Git usage can be ready without
  GitHub API access.
