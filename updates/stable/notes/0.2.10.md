# Git Tool 0.2.10

## Features

- Commit details can now restore a single committed file back into the working
  tree, including recreating files from deleted commit rows without staging the
  result. Normal commit diffs also expose per-hunk `Revert` actions for undoing
  one committed hunk in the working tree.
- Repository Access now displays and edits the raw repository-local remote URL
  instead of Git's globally rewritten `insteadOf` URL, preventing global
  transport aliases from looking like saved repo remotes.

## Fixes And Reliability

- Kept Monaco hunk action/header overlays out of the vertical scrollbar gutter,
  so commit hunk controls no longer cover the scrollbar.
- Made committed-hunk revert failures explain that the current file no longer
  matches the commit context instead of surfacing raw `git apply` patch errors.
- Persisted top-level sidebar section open/closed state and resized heights
  through repository switches and app restarts.
- Made sidebar section headers slightly more compact so branch panels leave more
  room for rows.
- Widened the sidebar resize rail so dragging the left panel is easier to hit.
- Added quieter lane-bound row washes so the checked-out branch remains
  visually distinct from manual commit clicks without using status-green.
- Strengthened normal commit graph row colors in both light themes so
  unselected lanes remain visible without matching the selected-row emphasis.
- Simplified the compare workspace refs card to use the branch pickers as the
  primary controls, while keeping commit refs visible when launched from commit
  graph actions, and made the middle switch control swap Base and Head.
- Added inline word-level highlights to Monaco diffs so changed spans stand out
  inside added and removed lines instead of relying only on full-line color.
- Refreshed the open working-tree diff after background status updates, so an
  unstaged file being inspected updates its existing text view in place when
  another process edits it.
- Made macOS patch preparation tolerate a renamed running app bundle, so a
  valid installed build such as `git-tool.app` can still match the release
  patch payload for `Git Tool.app`.
