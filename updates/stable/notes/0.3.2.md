# Git Tool 0.3.2

## Features

- Added repository-local **Protected changes** in the staging panel. Staged or
  unstaged files can be protected from row context menus, stay visible in a
  conditional section with an Unprotect action, and are guarded from accidental
  staging, commits, or bulk discard until unprotected again.
- Protected changes now remember their section collapse state and expose a
  `Clear stale protected` maintenance action when saved protected paths no
  longer appear in Git status.
- Made targeted discard actions status-aware: restoring deleted files uses
  neutral menu styling and runs without confirmation, while destructive
  file-content discards, including staged additions, stay in the danger style
  and require confirmation.
- Added explanatory tooltips to the interactive rebase row controls for
  reordering, editing commit messages, and dropping commits.
- Automatic assignment rules in Repository Access can now be edited in place,
  including their label, account, match kind, and match value.
- The title-bar repository switcher can now reveal the current repository from
  the closed-state right-click menu, and any recent repository from either its
  row icon or row right-click menu, without switching repositories first.
- Inline blame rows can now open the corresponding commit when a committed blame
  row is clicked, blend in commit-specific tone variants, and render newer rows
  brighter so same-author blame blocks remain easier to tell apart.
- Unified text diffs can now annotate the modified side from the editor context
  menu, keeping hunk headers, hidden ranges, and hunk action overlays aligned
  while using side-specific blame request keys.
- Made Escape close commit action dialogs consistently, including the
  interactive rebase dialog when focus is still on the graph.
- Commit graph context menus can now move a commit up or down one first-parent
  step on the checked-out branch, with the same clean-working-tree and replay
  conflict guards used by the existing history rewrite actions.
- Commit graph context menus now include a guided interactive rebase for the
  selected commit's child commits, covering reorder, drop, and edit-message
  actions with stale-plan and replay-conflict safeguards.
- Added branch-menu interactive rebase so the current branch can be reordered,
  edited, or dropped before replaying it onto another local branch.
- Updated branch rebase menu labels to name both branches, for example
  `Rebase feature/foo onto main...`.
- Commit graph branch, tag, and `+N` ref badges now expand to the right on hover
  so clipped ref names can be read directly over the graph lane.
- Added `Apply patch to <branch>...` on the checked-out branch context menu. It
  picks a `.patch` or `.diff` file from the system and applies it as unstaged
  working-tree changes.
- Branch checkouts that are blocked by staged, unstaged, or untracked changes
  can now stash those changes temporarily, checkout the target branch, and
  restore them. The first blocked checkout shows a rememberable confirmation,
  and the default can be changed in Settings > General > Git actions.
- Added LOCAL branch organization actions: hide, pin to left/top, and solo.
  Hidden and pinned branches persist per repository; hidden and solo states are
  clearable from the LOCAL header, and pinned branches show a small pin marker
  in the branch row.
- Diff inspection now prefetches nearby stash-file diffs too, and prefetched
  text diffs can opportunistically warm the mounted Monaco diff view-model cache
  so nearby file switches spend less time recomputing diff state.
- Working Tree, Commit Detail, Stash Detail, Pull Request files, Compare files,
  and nested Pull Request/Compare commit-file lists now start warming the first
  visible file diffs after their file lists render, using already loaded row
  metadata so panel-open prefetch does not repeat full status or name-status
  scans per file.
- The dev diagnostics dashboard now live-refreshes while open and can pause or
  resume that refresh from the title bar. Backend diagnostics now include
  sanitized Git command timings for failures, remote commands, and slow local
  commands, plus working-tree status stream timings and browser dev bridge
  warnings.
- The dev diagnostics dashboard now persists its selected tab and filters, shows
  timing summaries, follows the selected AI log run on refresh, can export a
  diagnostics bundle, and can compact the diagnostics and AI debug logs. Backend
  diagnostics now also cover provider/auth, OAuth, update, and AI entry points,
  and structured events can include trace IDs.
- Dev diagnostics now has a Performance tab with Go runtime memory, GC, and
  goroutine stats plus dashboard-local frontend heap, event-loop, RAF, and
  long-task samples, process RSS/CPU totals where supported, diff prefetch
  aggregate counters, Monaco diff view-model cache counters, and a backend
  performance snapshot in diagnostics bundle metadata.
- Diff prefetch diagnostics now split visible-list queue/fetch counts from
  click-neighbor prefetch counts so startup and panel-open warming can be tuned
  separately.
- Working Tree visible prefetch now keys its trigger to the first candidate
  window, reducing repeat warmups when lower-list status churn does not change
  the files likely to be clicked first. Prefetch fetch timings now record the
  prefetched path across file-list surfaces for follow-up diagnostics.
- Working Tree file selection now reuses a matching prefetch that is already in
  flight, avoiding duplicate Git diff requests when a row is clicked while its
  warmup is still finishing; diagnostics also record whether each prefetch
  result was stored or discarded as stale/unsupported.
- Switching from one open Working Tree diff to another now keeps the current
  inspector visible until the next file's diff and hunk layout are ready,
  avoiding the temporary empty or full-file loading surface between file clicks.
- Working Tree prefetch invalidation now records the clearing reason plus cache,
  pending queue, and in-flight counts so dev diagnostics can explain why a
  stored prefetch did not become a foreground cache hit.
- Selecting another Working Tree file no longer routes through the stash-clear
  reset path, so warmed diff cache entries survive normal file navigation.
- Dev diagnostics now records finer-grained working-tree diff foreground,
  working-tree row-click, remote-fetch lifecycle, and Monaco model lifecycle
  timing to help diagnose slow first file opens.
- Added an opt-in Playwright prefetch diagnostics workflow, with a build-time
  frontend diagnostics gate for production preview runs, to measure startup
  warmup, first-click latency, memory, and CPU before tuning prefetch caps.
- Added dev-only failure injection for repository, provider, update, stream,
  and guarded Git mutation paths so backend error states can be exercised in
  tests while verifying the user-facing Error Center, inline status, and state
  preservation behavior.

## Fixes

- Delayed merge-state saves now become harmless no-ops after a merge-class
  operation has already ended, avoiding repeated `no active merge` binding
  errors in the app log.
- Commit-file diff prefetch now stays pinned to the repository that queued it,
  so switching repositories cannot make a stale prefetch ask the new repository
  for an old file path.
- Removed the native Wails devtools keybindings that caused macOS accelerator
  parser errors when modifier-only key events reached the app.
- Kept the first-run setup Keys & Credentials tables on the same darker inset
  row surfaces as the Settings modal, and gave onboarding account detail panes
  enough bottom spacing at the end of their internal scroll.
- Replaced the browser sign-in device-code copy text button with the standard
  copy icon button and copied-state feedback.
- Anchored onboarding account edit actions in the account detail footer instead
  of letting them float over the scrolling form fields.
- App startup now reopens unfinished onboarding on the exact saved setup panel
  instead of restoring the repository workspace with only a Resume setup badge.
- Reopening unfinished onboarding no longer auto-fetches repository remotes
  before credentials or repository access are configured.
- Git action success toasts now wait for repository refresh and busy-state
  visuals to settle, so fetch/pull/push completion feedback appears in sync
  with graph and sidebar updates.
- Staging-panel actions now stay scoped to the repository that was visible when
  the user clicked them, preventing stale discard/stage/stash completions from
  surfacing Error Center messages after switching repositories.
