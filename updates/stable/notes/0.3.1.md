# Git Tool 0.3.1

## Features

- Refreshed the packaged app icon to match the simple Version & Updates
  Git mark while keeping standard 1024px macOS icon sizing.
- Staging and unstaging a file now advances the working-tree selection to the
  next row in the same source section, or the previous row when acting on the
  last item, in both normal and merge views.
- Staging rows now support arrow up/down single-row navigation plus
  shift+arrow up/down range selection from the keyboard, and selected
  applied-resolution rows keep an accent highlight on top of their green
  resolved state.
- Green staged applied-resolution rows are kept out of multi-row selected
  counts and bulk unstage selection because they can only be restored with
  their row-level Undo action. Keyboard range selection now shrinks when
  reversing shift+arrow direction.
- Submodule pointer diffs now include the pointed commit subject next to each
  old/new submodule commit hash when the submodule has that commit locally.
- Settings now opens on a General page with Display and Background tabs, where
  auto-fetch remotes can be turned off, limited to background-only fetching, or
  run while the app is open on a 1/3/5/10/15/30 minute interval.
- Changed image files can now be previewed in the shared file inspector with
  2-up, swipe, and onion-skin diff modes, and binary image file lists show KB
  size deltas instead of text line counts. File, Blame, and History views also
  show previewable raster images instead of binary text placeholders, with
  Blame preserving side-gutter attribution for images and Swipe offering a
  draggable center divider.
- The shared file inspector now shows line blame through `Annotate with Blame`
  in File mode, available from the toolbar and editor right-click menu, instead
  of a separate mode next to History. The inline blame gutter renders every row
  without gaps, appears only once rows are ready, virtualizes row rendering for
  smoother first paint, uses the app's graph-lane blues and purples for clearer
  author buckets, replaces native browser tooltips with an app-styled blame
  hover card, suppresses the Monaco fallback context menu, supports right-click
  `Close Annotations` from the editor or gutter, and highlights the matching
  code line on row hover. Edit mode can also show annotations from the editor
  context menu while the buffer is being changed; unchanged lines keep their
  attribution and inserted or changed lines render blank annotation rows. The
  app-owned Monaco context menus use the shared context-menu surface, keep
  normal Copy/Select All in read-only editors and Cut/Copy/Paste/Select All in
  editable editors, without exposing Cut/Paste on non-editing file views.
  Git-backed manual conflict review and result panels can now use the same
  right-click annotation flow, with the result pane preserving blame for
  unchanged selected source lines and leaving manually edited lines blank.
  Stash conflicts and the current related-file side of synthetic structural
  reviews now expose annotation when a trustworthy blame target can be resolved.
  Incoming conflict-side blame follows renames between `HEAD` and the operation
  ref; conflict-side refs that still do not contain the requested path return
  empty annotations instead of logging Git blame `exit status 128` binding
  errors. Manual result annotation gutters now reserve blank space for unfilled
  conflict-zone placeholders instead of painting blame rows through those gaps,
  and keep attribution after later conflict zones by avoiding a second result
  blame-alignment pass.
- The toolbar PR action now turns blue when open pull requests exist for the
  current branch.
  Left click still opens the full pull request workspace, while right click
  shows direct links to existing PRs plus create and refresh actions.
- Pull request target warnings now route missing provider-token states to the
  Accounts shortcut, matching expired and revoked token guidance.
- Structural conflict `Resolve structural` now keeps the structural
  target-finding pass separate and opens the related-file review without
  silently starting a normal text-resolution pass. Related-file structural
  reviews still render from one generated marker document instead of remapping
  zones back onto separate full source files, and AI Assist now explains what
  each AI action will do before the user starts the request.
- Normal conflict AI prompts now stay local-first around the provided conflict
  blocks and deterministic evidence, reducing broad repository exploration
  unless extra context is needed.
- Structural related-file refinement sends the text resolver the same generated
  synthetic marker document shown in the UI, plus instructions to preserve the
  current target shape while porting relevant incoming behavior. Large
  related-file reviews therefore resolve per synthetic zone instead of as one
  whole-file block when the user explicitly asks AI to draft/refine that result.
- Structural conflict AI Assist now exposes separate `Resolve structural` and
  `Full resolve` actions. Full resolve runs target discovery and then drafts a
  reviewable result for the discovered related-file review.
- Structural related-file reviews now avoid giant one-zone synthetic conflicts
  when Git sees the successor and obsolete original as one broad hunk. If the
  structural modifying diff is small, the backend builds focused zones around
  the actual incoming added lines.
- Structural related-file reviews now try that small incoming-only diff mapping
  before comparing the whole obsolete incoming file against the current
  successor, avoiding noisy review zones when the real incoming patch is just a
  few localized fixture or method changes.
- Structural synthetic review state now has an algorithm-versioned cache key,
  and markerless backend results remain markerless in the UI. This prevents
  stale or unmappable related-file reviews from reappearing as whole-file
  synthetic conflicts after reset/retry.
- Related-file synthetic reviews now keep the intent blocks that can be mapped
  into the current successor file even when later incoming blocks are
  legacy-only and no longer have a current counterpart.
- Structural related-file intent mapping now ignores indentation-only anchor
  drift, fixing successor test files whose fluent setter chains moved out of
  older nested test classes.
- Structural AI target selection now rejects weak test-to-main filename matches,
  so a deleted/modified test file does not open a production implementation as
  the related-file review target unless there is stronger move/history
  evidence.
- Conflict result attribution now recognizes indentation-only source-line
  drift, small token-preserving rewrites, and neighboring block context when a
  result line maps unambiguously to one side, so structural AI results keep the
  expected selected-line cues after adapting incoming code into the current
  related file.
- The frontend now builds a canonical conflict-resolution workspace before
  structural related-file AI handoff, checking that the visible synthetic marker
  zones and AI marker zones match before a prompt is sent.
- AI Assist suggestions can now be discarded from the suggestion card, clearing
  the saved AI draft and returning the result pane to manual review without
  changing the working tree.
- AI Assist refinement/re-resolve now keeps the conflict row/header activity
  indicator visible after the panel is closed, and clears it only when the
  refinement completes or is canceled. During refinement, the reset action is
  hidden so the active workflow has a single cancel button.
- AI Assist now shows `Reset` after an AI suggestion instead of a one-lane
  rerun action, returning the panel to the initial resolve choices.
- AI refinement now asks for a complete revised file result, so user
  instructions can update code outside the original conflict marker blocks.
- Structural related-file AI now rejects synthetic handoffs that cannot
  round-trip to the full current target file, and rejects dramatically short
  related-file results instead of showing partial source as a valid result.
- Synthetic structural review panes now refresh from source-content fingerprints,
  so updated current/incoming related-file content is shown immediately instead
  of waiting for an apply/undo reload.
- `Undo Apply` now removes the structural target staging created by Git Tool,
  preserves unrelated manual staged files, and refuses to run when the applied
  file or structural target has staged or unstaged edits made after apply.
- Structural related-file discovery now includes same-extension filename-token
  matches and treats `src/test` and `src/integrationTest` as compatible test
  locations, so successors like `DeliveryControllerV2IntegrationTests.java` ->
  `DeliveryControllerV2Tests.java` are offered to AI review.
- AI block conflict resolution now rejects obvious explanation prose inside
  code replacements, so model text like `keep the current...` fails the AI
  request instead of being inserted into the resolved source file.
- AI block conflict resolution now expands empty block replacements when the
  model clearly meant "keep current/ours" or "keep incoming/theirs", preventing
  those side-choice decisions from rendering as empty result zones.
- AI block conflict prompts now explicitly forbid web search/network access and
  no longer inherit full-file `resolvedContent` wording from the shared system
  prompt.
- AI block conflict validation now rejects comment-only replacements that are
  just decision prose, preventing results such as `/* keep the current... */`
  from being inserted into code.
- AI block conflict validation now rejects missing, duplicate, or unknown block
  IDs, and the AI debug log records each conflict document's resolution mode,
  marker-block count, and source sizes before the prompt is sent.
- AI block conflict validation now rejects suspiciously short replacements for
  very large code blocks, preventing a giant marker block from being collapsed
  into a small excerpt when the model cannot reproduce the unchanged body.
- Manual conflict review now supports explicit empty-side choices in the
  ours/theirs panes, so choosing the empty side of an add/delete-style zone
  clears the result-pane empty marker and persists as a resolved decision.
- AI result prefill now refuses suspiciously short full-file results for large
  conflicts, preserving the full manual-resolution document instead of
  collapsing it to a partial stub.
- Synthetic related-file structural reviews now keep shared lines inside large
  Git hunks as context instead of wrapping the entire hunk as one huge manual
  conflict, making moved/deleted large test files far more reviewable.
- The dev diagnostics dashboard now includes the dev-only AI debug log, with
  every line present in the log grouped by AI run, plus copy-path, refresh, and
  clear actions. Concurrent AI runs now stay separated even when their log lines
  are interleaved on disk. Selecting all inside the log now stays scoped to the
  selected run instead of the dashboard background. Production builds no longer
  create the AI debug log.

## Fixes

- Kept image diff `Swipe` and `Onion Skin` overlays aligned to the rendered
  image bounds instead of clipping against the whole inspector pane.
- Kept live sidebar and right-panel resizing smooth while the commit graph is
  visible by making the normal row accent paint cheaper without switching to a
  separate drag-only color style.
- Kept the working-tree diff inspector open when the selected file is staged or
  unstaged from its row action and no adjacent row exists.
- Reduced resize work in Monaco-backed conflict panels by coalescing duplicate
  editor layout passes, passing observed host dimensions directly into Monaco,
  and ignoring unchanged editor host resize callbacks.
- Disabled Monaco color decorators in file and diff inspectors so hash-like text
  no longer shows inline color swatches or opens color-picker affordances.
- Extended Monaco hunk-header tint through the scrollbar gutter with a
  translucent wash, so headers no longer stop short without obscuring the
  scrollbar.
- Standardized Settings panel surface colors so General, Shortcuts, and other
  settings pages share the same panel, row, and control hierarchy.
- Polished Settings section layouts so Version & Updates, AI setup, Accounts,
  Keys & Credentials, and Environments use calmer inset rows and app-blue
  generic selected states.
- Aligned the Repository Access automatic assignment rule metadata strip with
  the darker inset surfaces used by the current-repository details.
- Terminal tabs now keep a stable height while narrowing as more sessions are
  opened, use a searchable tab picker above the tab strip instead of a
  horizontal scrollbar once the strip is dense, and expose close-current,
  close-other, close-left, and close-right actions from the tab context menu.
- Terminal tab titles, order, count, and active selection now persist with the
  rest of the environment UI settings.
- Repository, environment, and terminal-tab dropdown lists now share a clearer
  app-accent scrollbar treatment.
- The pull request workspace branch picker now points from the head branch into
  the base branch, matching the actual PR direction.
- AI-resolved text conflicts now keep repeated or blank shared lines attached
  to the correct result-editor conflict zone instead of placing resolved lines
  just below the highlighted zone.
- Blank lines that belong to a conflict result zone now keep their purple zone
  highlight without requiring hover, so selected empty spacer lines remain
  visible in the result editor.
- Conflict rows now show the AI resolving spinner even while that conflict is
  the active file, so progress feedback appears immediately instead of only
  after switching to another conflict.
- AI conflict resolution now tracks in-flight requests per conflict path, so
  starting AI on another file no longer suppresses the first file's later
  result or timeout error.
- Structural related-target discovery now filters out files that are still
  unmerged, and backend validation rejects unresolved AI-selected targets. This
  prevents one structural conflict from opening a synthetic review against
  another unresolved conflict.
- Related-file structural reviews now use the original path's base-to-modifying
  diff as inferred change-intent evidence when available, filtering out
  synthetic zones that only come from broad shape differences between the
  current related file and the incoming original file.
- Full structural AI resolve now opens the related-file three-panel review with
  the drafted content result instead of stopping at target discovery.
- AI block conflict results now preserve the line break after a resolved block
  when the model omits the final newline, preventing the next shared source line
  from being glued onto the replacement.
- The file inspector editor now preserves CRLF line endings in Monaco text
  models, preventing CRLF files from being reset while typing in Edit mode.
- Structural related-file review panes now show the full current target and
  full incoming original contents on their respective sides while keeping the
  synthetic marker document for zone placement and result assembly.
