# Git Tool 0.3.3

## Features

- Added the first Vim mode foundation: Settings > Shortcuts now exposes Vim
  mode, leader-key, leader-hint, and relative row-number preferences; the app
  shows a global Vim mode indicator and supports safe normal-mode leader
  bindings for Settings, repository picker, terminal toggle, and panel
  navigation/toggles.

## Fixes

- Kept the blocked-checkout auto-stash toggle inside its confirmation row at
  narrow modal widths, and portaled inline-blame hover cards so gutter row
  popups stay anchored beside the hovered row across WebView/zoom setups.
- Removed the stray Settings footer close label and aligned the Settings close
  icon size with the app's other panel close buttons.
- Added the first contextual Vim binding route for the commit message input,
  including commit-message options, focus, AI generation, and normal-mode Enter
  submit through the existing readiness checks.
- Expanded safe Vim defaults with settings-section bindings, update settings,
  dev diagnostics, console window toggling, focus reset, and arrow aliases for
  window navigation.
- Added the Settings > Shortcuts Vim Bindings list for per-action sequence
  overrides, enable/disable toggles, reset controls, and conflict warnings.
- Kept prompt-level Vim defaults directly executable (`Space s`, `Space r`,
  `Space t`, `Space c`) and fixed normal-mode leader handling inside Settings,
  repository picker, and other popup surfaces so `Space w d` can close them.
- Added contextual Vim options for focused working-tree rows so `Space k`
  opens the existing staging context menu with its guarded actions and disabled
  states.
- Added basic normal-mode text-input motions for Vim mode: `h/j/k/l`, arrow
  aliases, `0`, `$`, `gg`, `G`, and `i`/`a`/`I`/`A` insert entry points.
- Added contextual Vim options for sidebar rows so `Space k` opens existing
  branch, tag, stash, and submodule menus with keyboard focus.
- Added contextual Vim options for focused commit graph rows so `Space k`
  opens the existing commit menu with keyboard focus.
- Added a graph-context Vim filter binding so `/` and `Space h f` focus the
  existing commit graph filter from a focused commit row.
- Added contextual Vim options for commit inspector file rows so `Space k`
  opens the existing changed-file menu with keyboard focus.
- Fixed AI provider cards so Settings keeps them as horizontal identity rows,
  and kept the Dev Diagnostics Performance tab from creating a horizontal
  scrollbar when diff-prefetch counters are visible.
- Widened the Dev Diagnostics dashboard so dense Timing and Performance filters
  have room before clipping.
- Added clearer spacing and a subtle divider between Dev Diagnostics filter
  groups.
- Kept the staging commit box visible when an empty Unstaged section is
  expanded, including with Vim mode's staging context wrapper mounted.
- Split Settings > Shortcuts into Keyboard and Vim tabs so Vim binding
  overrides no longer bury the normal app shortcuts.
- Added the first Vim command search surface for `Space Space`, seeded with
  safe action-registry commands and routed through the existing Vim action
  dispatcher.
- Made the Vim repository-picker binding useful before a repository is open by
  focusing the setup screen's visible `Open Repository` action.
- Added the first relative row-number rendering for staging file lists. The
  Keyboard-tab setting works without Vim mode and shows `0` on the focused
  row.
- Added direct Vim working-tree file leaves under `Space f` for opening,
  staging, unstaging, protecting/unprotecting, bulk stage/unstage, copying
  paths, and revealing the focused file through the existing staging actions.
- ArrowUp and ArrowDown now stay inside row lists such as the Working Tree,
  sidebar refs, commit graph, commit/stash file lists, PR/compare inspectors,
  file history, and picker popovers, revealing the active row instead of also
  scrolling the surrounding panel.
- Commit graph ArrowDown key repeat, including holding the key down, now keeps
  advancing through loaded rows, buffers repeat steps while older history is
  loading, keeps keyboard selection from toggling the detail closed during
  graph reloads, keeps focus on the stable graph listbox after row clicks, and
  triggers each older-history load window when navigating long histories.
- Pull request inspector files, commits, nested commit-file lists, and the
  conflict workspace now keep ArrowUp/ArrowDown on the active list/path
  selection while leaving Monaco and text-editing arrows alone.
- Added guarded Vim working-tree discard/delete leaves (`Space f d` /
  `Space f D`) that open the same confirmation prompts as the visible row
  actions.
- Sidebar stash rows can now be renamed from their context menu with an inline
  editor.
- Stash rename now swaps the inline editor to the new row label immediately
  while the backend refresh confirms the Git update.
- Clarified Vim leader hint prefix labels so subtrees read as actions, for
  example `Jump To Settings Section` instead of the ambiguous `open`.
- Removed the duplicate-looking default Vim `Space u u` Version & Updates
  alias; `Space o u` remains the settings-section jump and `Space u ...` is
  reserved for future update-flow actions.
- Added repository toolbar Vim leaves under `Space g` for fetch, pull, push,
  branch creation, pull-request workspace, stash, and pop stash. They reuse the
  existing toolbar callbacks instead of new Git paths.
- Added Vim leaves for the title-bar status popovers: `Space a a` opens
  Activity history and `Space e e` opens Error Center.
- Let the Vim hint panel grow with the viewport and removed the visible
  scrollbar strip from normal-sized leader menus.
- Kept the commit-message placeholder visible after whitespace-only drafts or
  accidental normal-mode leader spaces.
- Anchored the Vim mode indicator to the visible commit-message box whenever
  it exists, including after app restart, so it no longer sits on top of the AI
  commit button.
- Merged global Vim leader hints into contextual surfaces, so focusing the
  commit message still shows window, settings, repository, and other global
  routes alongside commit-message actions.
- Added terminal-tab Vim leaves while preserving raw shell input: focusing
  terminal chrome now supports `Space k` for the tab menu and
  `Space t n/f/r/d/o/H/L/j/k/q` for tab creation, search, rename, close
  variants, tab focus movement, and hiding the panel.
- Fixed sidebar section resizing so dragging a divider can keep borrowing
  space from other expanded sections after the immediate neighbor reaches its
  minimum height.
- Kept collapsed sidebar tabs from dragging past expanded sections that are
  already at their minimum height.
- Allowed collapsed sidebar headers between open sections to move when the
  pushed open section still has room to shrink.
- Working-tree Command/Ctrl-click selection now promotes a previously
  plain-clicked row into the multi-selection, so selected-file counts and
  context-menu labels include both files.
- Working-tree Command/Ctrl-click deselection now removes the row highlight
  when the focused anchor leaves the multi-selection.
- Working-tree Command/Ctrl-clicking the final selected file now clears the
  row focus so the original anchor does not reappear as selected.
- Selecting, closing, or minimizing around a normal staged, unstaged, or
  protected working-tree row during a merge conflict now keeps conflict
  auto-focus from pulling the selection back to the unresolved conflict list
  while the diff is open, and keeps the conflict workspace minimized across
  same-operation merge-state refreshes. Selecting a commit or stash while
  conflicts are minimized now keeps that detail panel open instead of bouncing
  back to the working tree.
- Added a read-only repository change monitor so external file edits, staging,
  commits, checkouts, pulls, and merge-state changes prompt debounced refreshes
  for staging, branch, graph, and conflict views.
- Removed the routine working-tree interval poll; staging now relies on
  watcher-triggered, explicit action, and focus/visibility refreshes, with a
  cheap backend metadata poll reserved for missed critical Git state changes.
